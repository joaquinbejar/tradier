## [0.2.0] - 2026-04-19

### Added

- **accounts**: `get_account_history`, `get_account_gain_loss`, and `get_account_orders` endpoints on both blocking and non-blocking clients, with typed pagination (`Page`, `Limit`), `EventType`, `GainLossSortBy`, `SortOrder` (promoted to `common`), and `IncludeTags`.
- **market_data**: 13 REST endpoints — `get_quotes` (GET + POST), `get_option_chains`, `get_option_strikes`, `get_option_expirations`, `get_lookup_option_symbols`, `get_historical_quotes`, `get_time_and_sales`, `get_etb_securities`, `get_clock`, `get_calendar`, `search_companies`, `get_lookup_symbol`. Co-located JSON schemas for the non-trivial responses.
- **fundamentals**: 7 REST endpoints — `get_company`, `get_corporate_calendars`, `get_dividends`, `get_corporate_actions`, `get_ratios`, `get_financials`, `get_statistics`. `FundamentalsEnvelope<T>` for the shared wire shape.
- **wssession**: typed market event response types (`MarketEvent` with `Quote`, `Trade`, `Summary`, `Timesale`, `Tradex`) and account event response types (`AccountEvent`). `MarketSession::event_stream` and `AccountSession::event_stream` return `Stream<Item = Result<…>>` of parsed events and keep reading past a single decode failure.
- **wssession**: `AccountSessionPayload` + `AccountSessionEvent` (`order`, `position`, `trade`, `fill`, `drop`) and `AccountSession::ws_stream`, mirroring the market session.
- **streaming**: HTTP chunked-transfer fallback (`streaming::http_stream::{market_events, account_events}`) reusing the pooled `reqwest::Client`.
- **common**: `Symbol` / `Symbols` newtypes and `SortOrder` promoted here so sibling endpoint modules can share them.
- Runnable examples for market data quotes, fundamentals company, and HTTP market streaming.

### Changed

- **build**: bump Rust edition to `2024`.
- **build**: `reqwest` gains the `stream` feature for chunked HTTP streaming; no new crates.
- **errors**: new variants `StreamDecodeError`, `ParseFloat`, `ParseInt`, `UnsupportedAccountEvent`, `MarketDataParseError` for precise failure surfaces.
- **accounts**: pagination counts standardised on `u32` across history, gain-loss, and orders.

### CI / Docs

- **ci**: `git-cliff` configuration and a tag-triggered GitHub Actions workflow that regenerates `CHANGELOG.md` on each `v*` tag push.
- **docs**: README sections on Conventional Commits, market data usage, fundamentals, and HTTP streaming.
